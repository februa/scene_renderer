# 受信信号生成ライブラリ 詳細設計書

## 1. 目的

本書は [基本設計書](./基本設計書.md) を受けて、最小構成実装のための型、メソッド、shape、例外、テスト観点を具体化する。

本詳細設計の対象は、2026-06-27 時点の最小構成実装である。背景雑音場生成とセンサ雑音生成の本体は次段階へ送るが、インターフェースと零出力の振る舞いは本書で固定する。

## 4. データ型詳細

### 4.1 Pose / Trajectory

```python
@dataclass(frozen=True)
class Pose:
    position_world: ArrayLike  # shape=(3,)
    heading_deg: float = 0.0
    pitch_deg: float = 0.0
    roll_deg: float = 0.0
```

- `position_world` は `float64` の `(3,)` に正規化する。
- 最小構成では回転に `heading_deg` だけを使う。
- `rotation_array_to_world()` は `ArrayFrame -> WorldFrame` の 3x3 行列を返す。
- `world_vector_to_array()` はアレイ投影で使うため、内積前の方向変換を担う。

```python
class Trajectory:
    def pose(self, t: float) -> Pose: ...
    def position(self, t: float) -> np.ndarray: ...
    def velocity(self, t: float) -> np.ndarray: ...
```

- `StaticPose.pose(t)` は `t` に依存せず同一 `Pose` を返す。
- `StaticPosition` は向きを持たない対象向けで、内部的には heading/pitch/roll を 0 に固定する。

### 4.2 AcousticSource / SourceComponent

```python
@dataclass(frozen=True)
class SourceComponent:
    spectrum: Spectrum
    envelope: Envelope
    amplitude: float | None = 1.0
    level_db: float | None = None
```

- `amplitude` と `level_db` の同時指定は禁止する。
- `amplitude_value` は描画直前に使う実効振幅である。
- `level_db` 指定時は `10 ** (level_db / 20)` に変換する。

```python
@dataclass(frozen=True)
class AcousticSource:
    trajectory: Trajectory
    components: list[SourceComponent]
```

- `from_position()` は `StaticPosition` に変換して保持する。
- `from_absolute_bearing()` は `WorldFrame` 上の単位ベクトルを使って位置を確定する。
- `from_relative_bearing()` は一度 `ArrayFrame` で方向を作り、`receiver_pose.array_vector_to_world()` で `WorldFrame` に変換する。

### 4.3 AmbientField

```python
@dataclass(frozen=True)
class AmbientField:
    spectrum: Spectrum
    amplitude: float = 0.0
    covariance: np.ndarray | None = None
    spatial_model: Any | None = None
```

- `covariance` は `(n_ch, n_ch)` を要求する。
- 非対称行列は `ValueError` とする。
- 固有値が負になる行列は `ValueError` とする。
- 最小構成では `spectrum` は保持のみで、零出力実装ではまだ直接使わない。

### 4.4 Receiver / ArrayGeometry

```python
@dataclass(frozen=True)
class Receiver:
    trajectory: Trajectory
    array: ArrayGeometry
```

```python
class ArrayGeometry:
    def positions(self) -> np.ndarray: ...
```

```python
@dataclass(frozen=True)
class LinearArray(ArrayGeometry):
    n_ch: int
    spacing: float
    axis: int = 0
    centered: bool = True
```

- `positions()` は `(n_ch, 3)` を返す。
- `axis=0/1/2` は `ArrayFrame` の `X/Y/Z` に対応する。
- `centered=True` の場合は座標平均が 0 になるように中心合わせする。

### 4.5 RenderedSource / PropagationPath / PropagatedSource

```python
@dataclass(frozen=True)
class RenderedSource:
    source: AcousticSource
    signal: np.ndarray          # shape=(n_sample,), dtype=complex64
    frequency: float | None = None
```

```python
@dataclass(frozen=True)
class PropagationPath:
    signal: np.ndarray          # shape=(n_sample,), dtype=complex64
    direction_world: np.ndarray # shape=(3,)
    delay: float
    gain: float
    path_type: str
    virtual_source_pos_world: np.ndarray | None = None
    frequency: float | None = None
```

```python
@dataclass(frozen=True)
class PropagatedSource:
    rendered_source: RenderedSource
    paths: list[PropagationPath]
```

- 最小構成では `paths` の要素数は 1 で、`path_type="direct"` のみを使う。
- `delay` は保持するが、時間波形そのものには反映しない。
- `virtual_source_pos_world` は将来の反射経路拡張時も同じ型で扱うために保持する。

## 5. レンダラ詳細

### 5.1 SceneRenderer

```python
class SceneRenderer:
    def __init__(
        self,
        contributors: list[MultiChannelContributor] | None = None,
        dtype: np.dtype | type = np.complex64,
    ) -> None: ...

    def render(
        self,
        scene: Scene,
        receiver: Receiver,
        axis_t: np.ndarray,
    ) -> np.ndarray: ...
```

処理順序は以下で固定する。

```text
1. axis_t 検証
2. fs 導出
3. contributor ごとの render 呼び出し
4. 多CH寄与の加算
5. 最終 dtype へ変換
```

`contributors=None` のとき、内部では以下を解決済みで保持する。

```python
[
    SourceFieldContributor(),
    AmbientFieldContributor(),
    SensorNoiseContributor(),
]
```

`dtype=float32` の場合、複素寄与が含まれていれば warning を出したうえで虚部を破棄する。

### 5.2 MultiChannelContributor

```python
class MultiChannelContributor:
    def render(
        self,
        scene: Scene,
        receiver: Receiver,
        axis_t: np.ndarray,
        fs: float,
    ) -> np.ndarray: ...
```

- 戻り値 shape は常に `(n_ch, n_sample)` とする。
- 局所寄与が複素で、背景・雑音が実数でも、`SceneRenderer` 側で最終的に合成可能な shape を返す。

### 5.3 SourceFieldContributor

```text
1. SourceRenderer.render(scene.sources, axis_t)
2. PropagationModel.propagate(...)
3. propagated_source ごとに ProjectorFactory.resolve(...)
4. ArrayProjector.project(...)
```

- `SourceRenderer` は音源ごとの時間波形生成を担当する。
- `PropagationModel` は `WorldFrame` のみを扱う。
- `ProjectorFactory` は音源や環境に応じた projector を返すが、最小構成では常に `NarrowbandPlaneWaveProjector` を返す。

### 5.4 SourceRenderer

```python
class SourceRenderer:
    def render(self, sources: list[AcousticSource], axis_t: np.ndarray) -> list[RenderedSource]: ...
```

- 最小構成では `ToneSpectrum` のみを受け付ける。
- 時間波形は `A * env(t) * exp(j 2π f t)` で生成する。
- 出力 dtype は `complex64` に正規化する。

### 5.5 FreeFieldPropagation

```python
class FreeFieldPropagation(PropagationModel):
    def propagate(...) -> list[PropagatedSource]: ...
```

- `receiver.trajectory.position(t0)` と `source.trajectory.position(t0)` を使って幾何を確定する。
- `t0` は `axis_t[0]` とする。
- `distance <= 0` は同一点配置エラーとして `ValueError` とする。
- `gain=1.0` とし、球面拡散は入れない。

### 5.6 NarrowbandPlaneWaveProjector

```python
class NarrowbandPlaneWaveProjector(SourceProjector):
    def project(...) -> np.ndarray: ...
```

処理内容は以下とする。

```text
1. direction_world を receiver_pose で direction_array に変換
2. tau_m = element_pos_array[m] dot direction_array / c
3. phase_m = exp(-j 2π f tau_m)
4. gain * phase_m * signal(t) を CH ごとに加算
```

- 最小構成では `frequency is None` を未実装として `NotImplementedError` とする。
- `delay` は使わず、CH 間の相対位相差だけを反映する。

### 5.7 ArrayProjector

```python
class ArrayProjector:
    def project(
        self,
        propagated_sources: list[PropagatedSource],
        projectors: list[SourceProjector],
        receiver: Receiver,
        environment: Environment,
        axis_t: np.ndarray,
        fs: float,
    ) -> np.ndarray: ...
```

- `projectors[i]` と `propagated_sources[i]` は 1 対 1 対応である。
- 数不一致は `ValueError` とする。
- projector ごとの多CH寄与を `(n_ch, n_sample)` 上で加算する。

### 5.8 AmbientFieldRenderer

- 最小構成ではゼロ信号を返す。
- 次段階では `covariance = L L^T` を満たす `L` を用い、`x = amplitude * (L @ w)` を実装する。

### 5.9 SensorNoiseGenerator

- 最小構成ではゼロ信号を返す。
- 次段階では `amplitude` を CH ごとの標準偏差として解釈し、独立ガウス雑音を生成する。

## 6. shape / dtype 規約

```text
axis_t.shape == (n_sample,)
array.positions().shape == (n_ch, 3)
RenderedSource.signal.shape == (n_sample,)
PropagationPath.signal.shape == (n_sample,)
contributor output shape == (n_ch, n_sample)
render output shape == (n_ch, n_sample)
```

```text
複素信号の基本 dtype: complex64
実信号のみの返り値 dtype: float32
座標・時間軸・方向ベクトル: float64 ベース
```

## 7. 例外仕様

### 7.1 入力検証

- `axis_t.ndim != 1` の場合は `ValueError`
- `axis_t.size == 0` の場合は `ValueError`
- `axis_t.size == 1` の場合は `ValueError`
- `axis_t` が厳密単調増加でない場合は `ValueError`
- `axis_t` が等間隔でない場合は `ValueError`
- `distance <= 0` の場合は `ValueError`
- `position_world.shape != (3,)` の場合は `ValueError`
- `LinearArray.n_ch <= 0` の場合は `ValueError`
- `LinearArray.spacing <= 0` の場合は `ValueError`
- `LinearArray.axis not in {0,1,2}` の場合は `ValueError`
- `AmbientField.covariance` が非正方の場合は `ValueError`
- `AmbientField.covariance` が非対称の場合は `ValueError`
- `AmbientField.covariance` が半正定値でない場合は `ValueError`
- `projectors` と `propagated_sources` の件数不一致は `ValueError`

### 7.2 未実装領域

- `ToneSpectrum` 以外を `SourceRenderer` に渡した場合は `NotImplementedError`
- `frequency is None` の経路を `NarrowbandPlaneWaveProjector` に渡した場合は `NotImplementedError`

## 9. 実装済みだが次段階で見直す点

- `SceneRenderer._validate_axis_t()` の許容誤差は現状 `atol=1e-12` 固定である。
- `AmbientFieldRenderer` と `SensorNoiseGenerator` は API のみ先行で、中身は零出力である。
- `PropagationPath.delay` は保持のみで、時間シフト処理に未接続である。
- `ProjectorFactory` は常に単一 projector を返す簡略版である。
- `ToneSpectrum.evaluate()` は現状のレンダリング経路では未使用であり、将来の周波数領域拡張で意味を持たせる。
