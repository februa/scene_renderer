## 4. データ型詳細

### 4.1 Pose / Trajectory

```python
@dataclass(frozen=True)
class Pose:
    position_world: ArrayLike  # shape=(3,)
    heading_deg: float = 0.0
    pitch_deg: float = 0.0
    roll_deg: float = 0.0
```

- `position_world` は `float64` の `(3,)` に正規化する。
- 最小構成では回転に `heading_deg` だけを使う。
- `rotation_array_to_world()` は `ArrayFrame -> WorldFrame` の 3x3 行列を返す。
- `world_vector_to_array()` はアレイ投影で使うため、内積前の方向変換を担う。

```python
class Trajectory:
    def pose(self, t: float) -> Pose: ...
    def position(self, t: float) -> np.ndarray: ...
    def velocity(self, t: float) -> np.ndarray: ...
```

- `StaticPose.pose(t)` は `t` に依存せず同一 `Pose` を返す。
- `StaticPosition` は向きを持たない対象向けで、内部的には heading/pitch/roll を 0 に固定する。

### 4.2 AcousticSource / SourceComponent

```python
@dataclass(frozen=True)
class SourceComponent:
    spectrum: Spectrum
    envelope: Envelope
    amplitude: float | None = 1.0
    level_db: float | None = None
```

- `amplitude` と `level_db` の同時指定は禁止する。
- `amplitude_value` は描画直前に使う実効振幅である。
- `level_db` 指定時は `10 ** (level_db / 20)` に変換する。

```python
@dataclass(frozen=True)
class AcousticSource:
    trajectory: Trajectory
    components: list[SourceComponent]
```

- `from_position()` は `StaticPosition` に変換して保持する。
- `from_absolute_bearing()` は `WorldFrame` 上の単位ベクトルを使って位置を確定する。
- `from_relative_bearing()` は一度 `ArrayFrame` で方向を作り、`receiver_pose.array_vector_to_world()` で `WorldFrame` に変換する。

### 4.3 AmbientField

```python
@dataclass(frozen=True)
class AmbientField:
    spectrum: Spectrum
    amplitude: float = 0.0
    covariance: np.ndarray | None = None
    spatial_model: Any | None = None
```

- `covariance` は `(n_ch, n_ch)` を要求する。
- 非対称行列は `ValueError` とする。
- 固有値が負になる行列は `ValueError` とする。
- 最小構成では `spectrum` は保持のみで、零出力実装ではまだ直接使わない。

### 4.4 Receiver / ArrayGeometry

```python
@dataclass(frozen=True)
class Receiver:
    trajectory: Trajectory
    array: ArrayGeometry
```

```python
class ArrayGeometry:
    def positions(self) -> np.ndarray: ...
```

```python
@dataclass(frozen=True)
class LinearArray(ArrayGeometry):
    n_ch: int
    spacing: float
    axis: int = 0
    centered: bool = True
```

- `positions()` は `(n_ch, 3)` を返す。
- `axis=0/1/2` は `ArrayFrame` の `X/Y/Z` に対応する。
- `centered=True` の場合は座標平均が 0 になるように中心合わせする。

### 4.5 RenderedSource / PropagationPath / PropagatedSource

```python
@dataclass(frozen=True)
class RenderedSource:
    source: AcousticSource
    signal: np.ndarray          # shape=(n_sample,), dtype=complex64
    frequency: float | None = None
```

```python
@dataclass(frozen=True)
class PropagationPath:
    signal: np.ndarray          # shape=(n_sample,), dtype=complex64
    direction_world: np.ndarray # shape=(3,)
    delay: float
    gain: float
    path_type: str
    virtual_source_pos_world: np.ndarray | None = None
    frequency: float | None = None
```

```python
@dataclass(frozen=True)
class PropagatedSource:
    rendered_source: RenderedSource
    paths: list[PropagationPath]
```

- 最小構成では `paths` の要素数は 1 で、`path_type="direct"` のみを使う。
- `delay` は保持するが、時間波形そのものには反映しない。
- `virtual_source_pos_world` は将来の反射経路拡張時も同じ型で扱うために保持する。

## 5. レンダラ詳細

### 5.1 SceneRenderer

```python
class SceneRenderer:
    def __init__(
        self,
        contributors: list[MultiChannelContributor] | None = None,
        dtype: np.dtype | type = np.complex64,
    ) -> None: ...

    def render(
        self,
        scene: Scene,
        receiver: Receiver,
        axis_t: np.ndarray,
    ) -> np.ndarray: ...
```

処理順序は以下で固定する。

```text
1. axis_t 検証
2. fs 導出
3. contributor ごとの render 呼び出し
4. 多CH寄与の加算
5. 最終 dtype へ変換
```

`contributors=None` のとき、内部では以下を解決済みで保持する。

```python
[
    SourceFieldContributor(),
    AmbientFieldContributor(),
    SensorNoiseContributor(),
]
```

`dtype=float32` の場合、複素寄与が含まれていれば warning を出したうえで虚部を破棄する。

### 5.2 MultiChannelContributor

```python
class MultiChannelContributor:
    def render(
        self,
        scene: Scene,
        receiver: Receiver,
        axis_t: np.ndarray,
        fs: float,
    ) -> np.ndarray: ...
```

- 戻り値 shape は常に `(n_ch, n_sample)` とする。
- 局所寄与が複素で、背景・雑音が実数でも、`SceneRenderer` 側で最終的に合成可能な shape を返す。

### 5.3 SourceFieldContributor

```text
1. SourceRenderer.render(scene.sources, axis_t)
2. PropagationModel.propagate(...)
3. propagated_source ごとに ProjectorFactory.resolve(...)
4. ArrayProjector.project(...)
```

- `SourceRenderer` は音源ごとの時間波形生成を担当する。
- `PropagationModel` は `WorldFrame` のみを扱う。
- `ProjectorFactory` は音源や環境に応じた projector を返すが、最小構成では常に `NarrowbandPlaneWaveProjector` を返す。

### 5.4 SourceRenderer

```python
class SourceRenderer:
    def render(self, sources: list[AcousticSource], axis_t: np.ndarray) -> list[RenderedSource]: ...
```

- 最小構成では `ToneSpectrum` のみを受け付ける。
- 時間波形は `A * env(t) * exp(j 2π f t)` で生成する。
- 出力 dtype は `complex64` に正規化する。

### 5.5 FreeFieldPropagation

```python
class FreeFieldPropagation(PropagationModel):
    def propagate(...) -> list[PropagatedSource]: ...
```

- `receiver.trajectory.position(t0)` と `source.trajectory.position(t0)` を使って幾何を確定する。
- `t0` は `axis_t[0]` とする。
- `distance <= 0` は同一点配置エラーとして `ValueError` とする。
- `gain=1.0` とし、球面拡散は入れない。

### 5.6 NarrowbandPlaneWaveProjector

```python
class NarrowbandPlaneWaveProjector(SourceProjector):
    def project(...) -> np.ndarray: ...
```

処理内容は以下とする。

```text
1. direction_world を receiver_pose で direction_array に変換
2. tau_m = element_pos_array[m] dot direction_array / c
3. phase_m = exp(-j 2π f tau_m)
4. gain * phase_m * signal(t) を CH ごとに加算
```

- 最小構成では `frequency is None` を未実装として `NotImplementedError` とする。
- `delay` は使わず、CH 間の相対位相差だけを反映する。

### 5.7 ArrayProjector

```python
class ArrayProjector:
    def project(
        self,
        propagated_sources: list[PropagatedSource],
        projectors: list[SourceProjector],
        receiver: Receiver,
        environment: Environment,
        axis_t: np.ndarray,
        fs: float,
    ) -> np.ndarray: ...
```

- `projectors[i]` と `propagated_sources[i]` は 1 対 1 対応である。
- 数不一致は `ValueError` とする。
- projector ごとの多CH寄与を `(n_ch, n_sample)` 上で加算する。

### 5.8 AmbientFieldRenderer

- 最小構成ではゼロ信号を返す。
- 次段階では `covariance = L L^T` を満たす `L` を用い、`x = amplitude * (L @ w)` を実装する。

### 5.9 SensorNoiseGenerator

- 最小構成ではゼロ信号を返す。
- 次段階では `amplitude` を CH ごとの標準偏差として解釈し、独立ガウス雑音を生成する。

## 11. 2026-07-08 コード品質見直し結果

### 11.1 見直し方針

AGENTS.md の規約に合わせ、公開クラス・公開メソッドの docstring、shape / axis / 単位コメント、信号処理式の対応コメント、Pylance / Pyright の型推論を重点的に見直した。

本リポジトリはビームフォーミング本体ではなく、後段ビームフォーミングへ渡す受信信号 `x[ch, t]` を生成するライブラリである。そのため Beamforming Evaluation の観点では、今回の対象は `fixed_beam_single_source` 評価の前段入力に相当する CH 間位相差、時間軸、dB 振幅換算、shape 契約の妥当性確認に限定した。BL / FRAZ / BTR や SLC 指標は、本リポジトリ内に該当する beamformer / SLC 実装がないため今回の必須評価対象外とした。

### 11.2 実装修正

- `np.ndarray` の裸型注釈を避け、NumPy 配列を扱う公開 API は `numpy.typing.NDArray[Any]` を使う方針へ揃えた。
- `SceneRenderer.render()` の出力契約を `x[ch, t]`、shape `[n_ch, n_sample]` として明記し、contributor 出力 shape の検証意図をコメントした。
- `NarrowbandPlaneWaveProjector` に、`tau[ch] = r[ch] dot direction_array / c` と `exp(-j 2π f tau[ch])` の対応、ならびに broadcasting 後の shape `[n_ch, n_sample]` をコメントした。
- `FreeFieldPropagation` に、`distance / c` の単位、同一点配置で到来方向が定義できない理由、球面拡散をまだ入れない理由をコメントした。
- `AmbientField` の共分散検証に、対称性・半正定値性・`1e-12` 許容値の意味をコメントした。
- `SensorNoiseGenerator` は将来の標準偏差指定として `amplitude` を検証し、負値または 2 次元以上の配列を拒否するようにした。
- `SceneRenderer` は `axis_t` に NaN / inf が含まれる場合、`fs` と位相回転が定義できないため `ValueError` とするようにした。
- PEP 561 の typed package 表明に合わせて `scene_renderer/py.typed` を追加し、`pyproject.toml` の package-data に含めた。
- 開発時の型検証を再現できるよう、`pyright` を dev optional dependency に追加した。

### 11.3 維持した設計判断

- `AmbientFieldRenderer` と `SensorNoiseGenerator.generate()` は、現在も shape `[n_ch, n_sample]` のゼロ信号を返す。統計モデル、PSD 基準、乱数再現性が未確定のまま非ゼロ雑音を返すと評価結果の基準が曖昧になるためである。
- `PropagationPath.delay` は保持のみで、時間波形への絶対遅延はまだ反映しない。今回の最小構成では CH 間相対位相差の検証を優先する。
- `ProjectorFactory` は現在も常に `NarrowbandPlaneWaveProjector` を返す。`SourceRenderer` が `ToneSpectrum` のみを扱う設計と整合しているためである。

## 12. 2026-07-08 広帯域音源設計と実装

### 12.1 基本方針

`scene_renderer` は stateful streaming renderer ではなく、呼び出しごとに渡された `axis_t` 区間の `x[ch, t]` を生成する stateless block renderer として扱う。

chunk 運用時に呼び出し側は絶対 sample index に対応する `axis_t` を渡す。`scene_renderer` は内部で乱数状態を持ち越さず、広帯域ノイズについても `noise_seed` と global sample index から決定論的に白色系列を生成する。このため、同じ絶対時刻範囲であれば、chunk 分割に依存せず同じ基準音源信号を得る。

ProjectorFactory は狭帯域・広帯域では分岐しない。Factory の責務は波面モデルの選択であり、現在は常に `PlaneWaveProjector` を返す。狭帯域 tone と広帯域信号の違いは、同じ `PlaneWaveProjector` の内部で遅延適用方式として扱う。

### 12.2 NoiseSpectrum

広帯域ノイズの帯域や色付けは `NoiseSpectrum` で表す。`NoiseSpectrum.evaluate(freq_axis)` は FIR 設計用の目標振幅応答を返す。

```text
freq_axis.shape == [n_bin]
abs(NoiseSpectrum.evaluate(freq_axis)).shape == [n_bin]
```

追加したノイズ用 Spectrum は以下である。

```text
BandLimitedNoiseSpectrum
  f_low_hz から f_high_hz まで振幅 1、それ以外 0 の平坦帯域を表す。

PinkNoiseSpectrum
  PSD が 1/f になるよう、振幅応答 1/sqrt(f) を返す。

CustomNoiseSpectrum
  周波数・振幅テーブルを線形補間し、任意の目標振幅応答を返す。
```

`Spectrum` は周波数特性の定義だけを持つ。白色系列生成、FIR 係数設計、畳み込み、chunk 境界処理は `SourceRenderer` の責務である。

### 12.3 SourceComponent のノイズ指定

`NoiseSpectrum` を持つ `SourceComponent` では、以下を追加指定する。

```python
noise_seed: int | None = None
noise_filter_length: int = 257
```

`NoiseSpectrum` の場合は `noise_seed` を必須とする。呼び出し順に依存する RNG 状態ではなく、`noise_seed` と global sample index から直接白色系列を作るためである。

`noise_filter_length` は奇数かつ 3 以上とする。FIR は中心 sample を持つ線形位相 FIR として扱うため、偶数長は許可しない。

### 12.4 SourceRenderer の広帯域生成

`SourceRenderer.render()` は以下の引数を受ける。

```python
render(sources, axis_t, fs=None, sample_margin=0)
```

`sample_margin` は projector が後段の fractional delay FIR に必要とする halo サンプル数である。`SourceRenderer` は元の `axis_t` の前後に `sample_margin` 分を追加した拡張時間軸を作り、`RenderedSource.signal` へ halo 付き信号を格納する。

`RenderedSource.center_slice` は、元の `axis_t` に対応する中央区間を表す。

```text
RenderedSource.signal.shape == [n_sample + 2 * sample_margin]
RenderedSource.center_slice == slice(sample_margin, sample_margin + n_sample)
```

広帯域ノイズ生成は以下の順で行う。

```text
1. axis_t[0] と fs から global sample index を復元する。
2. sample_margin と noise_filter_length / 2 を考慮した白色系列 index 範囲を決める。
3. SplitMix64 により、sample index と noise_seed から標準正規白色系列を決定論的に生成する。
4. NoiseSpectrum.evaluate(freq_axis) の振幅応答から FIR を周波数サンプリング法で設計する。
5. FIR を Hann 窓で有限長化する。
6. sum(h^2)=1 となるよう FIR を正規化する。
7. 白色系列へ valid 畳み込みし、期待 RMS=1 の広帯域ノイズを得る。
8. SourceComponent.amplitude_value と Envelope を掛ける。
```

`sum(h^2)=1` 正規化は、入力白色系列の分散が 1 のとき FIR 出力の期待 RMS を 1 に保つためである。chunk ごとの実測 RMS で正規化すると chunk 分割依存になるため、時間波形側ではなく FIR 係数側で正規化する。

### 12.5 PlaneWaveProjector

`NarrowbandPlaneWaveProjector` は `PlaneWaveProjector` へ置き換えた。後方互換のため旧名称は subclass として残す。

`PlaneWaveProjector` は、各 path について以下を計算する。

```text
direction_array = receiver_pose.world_vector_to_array(direction_world)
tau[ch] = element_pos_array[ch] dot direction_array / c
```

`path.frequency is not None` の場合は、狭帯域 tone として既存の位相回転を使う。

```text
x[ch, t] = gain * s[t] * exp(-j 2π frequency tau[ch])
```

`path.frequency is None` の場合は、広帯域信号として fractional delay FIR を使う。

```text
tau_relative[ch] = tau[ch] - min(tau)
delay_samples[ch] = tau_relative[ch] * fs
x[ch, t] = gain * fractional_delay(signal, delay_samples[ch])
```

`min(tau)` を基準にする理由は、`scene_renderer` が絶対伝搬遅延ではなく CH 間相対遅延を再現する renderer であり、全 CH の遅延を非負にして有限 block の前方 halo で処理できるようにするためである。共通遅延は捨てるが、CH 間相対遅延は保持される。

fractional delay FIR は windowed-sinc である。`PlaneWaveProjector.required_sample_margin()` は、FIR 半長とアレイ開口から見積もった最大 CH 間遅延を足した halo を返す。

```text
required_margin = filter_length // 2 + ceil(max_pairwise_element_distance / c * fs)
```

`SourceFieldContributor` はこの margin を `SourceRenderer` へ渡す。これにより、projector は state を持たずに、chunk 境界で必要な前後サンプルを各呼び出し内で再生成できる。

### 12.6 検証項目

追加検証は以下である。

```text
BandLimitedNoiseSpectrum の通過帯域 mask が正しいこと。
PinkNoiseSpectrum が PSD 1/f に対応する振幅 1/sqrt(f) を返すこと。
NoiseSpectrum で noise_seed 未指定を拒否すること。
広帯域 SceneRenderer 出力が full block と chunk 連結で一致すること。
既存 tone の CH 間位相差テストが引き続き通ること。
```

Beamforming Evaluation の観点では、今回も beamformer 本体ではなく入力生成の変更であるため、評価対象は `fixed_beam_single_source` の前段入力整合に限定する。確認対象は、CH 間相対遅延、入力波形の帯域、RMS レベル正規化、NaN/inf 不在、shape `[n_ch, n_sample]` の維持である。
